# 第I部インプット資料

## 目的

第I部向けのインプット資料を、`単元別` かつ `1問1答` で蓄積する。

この資料の主眼は次の 3 つです。

1. 問題文を見て、何を答えるべきかを即座に判断できること
2. 問題の背景にある論点を理解したうえで、基準答案を再現できること
3. 同じ論点の年度違いを同一テーマとして管理できること

## 1カードの構造

各カードは、Markdown 上で次の順番に記載する。

1. `問題`
2. `ざっくり説明` (任意だが原則入れる)
3. `要するに何か`
4. `問題意識`
5. `詳細説明`
6. `基準答案`

## 書式

```md
## CARD FP-XX-YYYY-NNN

- source: 2023 生保2 問2(2)
- tags: first_part rm 2023
- priority: A

### 問題
ここに問題文または問題文の実用的な要約を書く。

### ざっくり説明
まず直感でつかむための説明を 3-8 行で書く。

### 要するに何か
採点者が何を取りに来ている問題かを 2-5 行で書く。

### 問題意識
なぜその論点が重要か、どこが混乱しやすいかを書く。

### 詳細説明
基準答案の各行の行間を埋める説明を書く。

### 基準答案
Anki で再生したい本文を書く。
```

## 記述ルール

- `問題` は、Anki の front に出す前提で書く
- `基準答案` は、原則として 1 カード 1 答案にする
- `基準答案` は、原則として `過去問の解答` があるならその文言を優先して残す。過去問の解答がない場合やまとめた方がよい場合でも、`既存ノート` `法令` `一次資料` の文言をできるだけ崩さない
- `ざっくり説明` は、その論点を初めて見る人でも入口をつかめるように書く
- `要するに何か` は、答案ではなく自分用の理解補助
- `問題意識` は、制度趣旨、実務上の悩み、頻出の引っかけを明示する
- `詳細説明` は、`基準答案` の各行がなぜそうなるかを、前提概念から行間なく説明する
- 長すぎる答案は、段落単位または観点単位でカードを分割する
- `study/first_part/` では、強調のための `太字` に依存しない。強調したい場合でも、基本は文章の流れで読ませる

## `ざっくり説明` `要するに何か` `問題意識` `詳細説明` の書き方

ここは短い要約では足りない。丸暗記を安定させるため、次の順番で説明する。

1. `ざっくり説明` で、まず直感的な入口を作る
2. そもそも何を守りたいのか
3. そのために何を管理・評価したいのか
4. なぜその論点が試験で問われるのか
5. 受験生がどこで混乱しやすいのか
6. 何がきっかけで、何を通じて、どの指標がどう動くのか
7. その動きが会計・規制・資金繰り・経営判断にどう波及するのか
8. 可能なら `小さな数値例` で、何がどう動くのかを具体化する
9. `詳細説明` では、`基準答案` の文言をなぞりながら、各行の前提・因果・制度趣旨を順に埋める

必要に応じて、カード本文または単元冒頭に `前提概念` を置く。

例:

- 現在価値とは何か
- デュレーションとは何か
- 経済価値ベースのサープラスとは何か
- 規制上、何をリスクとして見ているのか

悪い例:

- `イミュナイゼーションの3条件を問う小問`

良い例:

- `生命保険会社は長い負債を抱えており、金利が動くたびに資産と負債の価値のズレが広がる。そこで、金利変動によって純資産やサープラスが大きく振れないよう、資産側を負債に合わせにいく。イミュナイゼーションの3条件は、その最も基本的な整合条件を聞いている。`

さらに良い例:

- `生命保険会社は、遠い将来に払う約束が多いので、負債の価値は金利変動に強く反応しやすい。たとえば資産100・負債90・純資産10で出発しても、負債のデュレーションが20年、資産のデュレーションが5年なら、金利低下時に負債の価値だけ大きく増えて純資産が急減する。このズレを小さくする基本条件がイミュナイゼーションである。`

## 行間を残さないための原則

- `ズレる` と書いたら、何と何がどうズレるのかまで書く
- `悪化する` と書いたら、どの指標がどちら向きに動くのかまで書く
- `問題になる` と書いたら、なぜ経営上まずいのかまで書く
- `規制上まずい` と書いたら、どの比率・区分・監督対応に波及するのかまで書く
- `流動性が苦しい` と書いたら、どの資金流出が起点で、どの資産をどう現金化する必要があるのかまで書く
- `デュレーション` `コンベクシティ` `ESR` `解約オプション` のような専門語を使うときは、その語自体を 1-3 文で説明する
- 可能なら `金利が1%下がると...` のような簡単な方向性の例を入れる
- その説明に前提概念が必要なら、そこで止まらず `前提概念そのもの` も説明する

## 初学者に説明し切る基準

- 読み手を `その論点を初めて学ぶ人` とみなし、専門語を使ったらその場で噛み砕く
- `なぜそれが必要なのか` を、制度趣旨だけでなく `それが無いと何が壊れるか` まで書く
- 会計論点なら、`何の金額をいつ収益化し、いつ負債に戻し、いつ費用化するか` を時系列で示す
- 規制論点なら、`規制当局が何を壊れ方として見ているのか`、`どの比率・区分・監督対応に跳ねるのか` まで書く
- 数値論点なら、`古いノートの値` と `現行法の値` がズレやすいので、必ず現行一次資料を確認する
- 文章は乾いた要約で終わらせず、`ここを外すと危ない` `ここが会計の急所` という熱量を持って書く

## 最低チェックリスト

各カードの `ざっくり説明` `要するに何か` `問題意識` `詳細説明` は、少なくとも次を満たす。

1. `ざっくり説明` を読むだけで、大意がつかめる
2. `何を守りたいか` が書かれている
3. `何が壊れ方の起点か` が書かれている
4. `途中の伝達経路` が書かれている
5. `最後にどの指標が動くか` が書かれている
6. `なぜ実務上困るか` が書かれている
7. `受験上の引っかけ` が書かれている
8. 必要な `前提概念` が同じページ内で説明されている
9. 少なくとも 1 つは `抽象語を具体例に落とした説明` が入っている
10. `詳細説明` を読むと、`基準答案` の各行に「なぜそうなるか」が対応づけられる

この条件を満たしていない説明は、`意味は合っていても暗記用としては未完成` とみなす。

## ソースの使い方

説明欄は、自分の言葉でかみ砕いて書く。ただし、根拠は一次資料またはそれに準ずる資料に置く。

優先順位:

1. 教科書
2. 法令・監督指針・実務基準
3. IAIS / IAA / 金融庁などの一次資料
4. アクチュアリー会の研究会資料・報告書
5. 補助的な論文・レポート

各単元ファイルでは、必要に応じて末尾に `参考にした資料` を置いてよい。

説明の目的は、読んだときに `なるほど、そういう意味か` まで到達させることであり、単に抽象語を言い換えることではない。

## 回答案の補完方針

WB の出典問題に対して原問や手元解答が不足する場合、次の順に補完する。

1. 既存ノート
2. 教科書 PDF
3. 類似過去問
4. 公式・準公式の外部ソース
5. 上記を踏まえた再構成答案

外部ソースを使う場合でも、最終的な `要するに何か` と `問題意識` は必ず自分用にかみ砕いて書く。

## 推奨運用

- まず `note/単元別マークダウン/` を基礎にする
- 次に `2021-2025` を追補する
- 最後に WB にない最近論点を厚くする

現時点の優先テーマは次のとおり。

- 生命保険会計
- 契約者配当
- ソルベンシー
- リスク管理・ALM
