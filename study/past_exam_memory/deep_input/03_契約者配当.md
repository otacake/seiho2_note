# 03_契約者配当

## この単元の芯

契約者配当は、「会社が儲かったから契約者へ返す」という単純な話ではない。生命保険の保険料は、将来の死亡率、利率、事業費を見込んで決める。その見込みには、安全のための余裕が入る。長い時間が経つと、実際の死亡率、運用利回り、事業費が見えてくる。その結果、見込みより余った部分が生じる。これを、誰に、いつ、どの程度返すかが契約者配当である。

ただし、余ったように見えるものをすぐ返せばよいわけではない。生命保険では、いま残っている契約者と、これから消滅する契約者が同じ群団の中にいる。消滅する契約者へ厚く返しすぎると、残る契約者の支払能力が薄くなる。逆に内部留保を厚くしすぎると、剰余に貢献した契約者へ返していないことになる。契約者配当の答案で「公正・衡平」「健全性」「アセット・シェア」が並ぶのは、この緊張関係を説明するためである。

保険計理人の実務基準でも、配当確認は単なる計算チェックではなく、代表契約、ネット・アセット・シェア、将来の健全性などを通じて見ていく構造になっている。これは、配当が一年度の利益処分ではなく、長期契約群団の中での還元だからである。

海外の実務資料でも同じ芯が出る。ASOPの配当実務基準は、配当フレームワークや配当要素の決定がアクチュアリー業務の対象になることを示している。SOAの配当メカニクス資料も、伝統的な有配当生命保険で、剰余をどう配当へ結びつけるかを扱う。日本の試験答案とは制度名が違っても、「経験差をどう契約者へ戻すか」「契約者群団の中で不公平を作らないか」という問題意識は共通している。

## 暗記事項は何を言っているか

「契約者配当を行う理由」は、安全性の原則、経験料率、保険料率の調整、競争上の手段、インフレ対応などで出る。ここは丸暗記しやすいが、芯はひとつである。保険料決定時点では分からなかったことを、あとから実績で調整する仕組みである。

「利源別配当方式」は、剰余を利差、死差、費差などの発生源に分けて配当する考え方である。保険料が複数の基礎率で作られている以上、剰余もどこから出たかを分けて見る方が、契約者間の納得感を作りやすい。

「アセット・シェア」は、個々の契約または代表契約が、過去にどれだけ資産を積み上げてきたかを見る考え方である。利源別配当が単年度の発生源に強いのに対し、アセット・シェアは累積的な貢献度を見る。特別配当や消滅時配当の話でアセット・シェアが出るのは、契約が消えるタイミングで、累積した持ち分をどこまで返すかが問題になるからである。

「通常配当」と「特別配当」は、毎年の経験差を返すのか、長期に蓄積した剰余や含み益を特定のタイミングで返すのかの違いとして読む。特別配当は、返す側の契約者だけを見ると多い方がよい。しかし、返した後に残る契約者群団の健全性が落ちるなら、公正・衡平ではない。

## 具体的に見る

平準払終身保険で、予定利率より実際の運用利回りが高かったとする。このとき利差益が出る。毎年配当タイプなら、契約者へ利差配当を返す発想になる。しかし、保険期間は長く、将来の金利低下や死亡率悪化もありうる。足元の利差益を全部返せば、今年の契約者は喜ぶが、将来の支払余力が弱くなる。

一時払終身保険では、契約時に保険料が一括で入り、資産運用との対応を取りやすい。平準払終身保険では、将来の保険料がこれから入るため、将来の運用環境の影響を受ける。同じ「利差益」でも、商品特性によって配当の考え方が変わる。2024年の過去問で商品特性、予定利率差、キャピタルゲイン還元が問われているのは、まさにこの部分である。

## 読み方のコツ

配当の答案は、次の三つを必ず分ける。

1. 剰余はどこから出たか。
2. その剰余に誰がどれだけ貢献したか。
3. 返した後、残る契約者群団は安全か。

この三つを同時に見ると、「公正・衡平」が単なるきれいな言葉ではなくなる。公平に返すことと、会社を安全に残すことは、同じ方向を向くこともあれば衝突することもある。答案はその衝突を、利源別配当、アセット・シェア、内部留保、将来収支分析でさばいている。
