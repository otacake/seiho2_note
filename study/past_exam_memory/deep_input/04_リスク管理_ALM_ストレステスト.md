# 04_リスク管理_ALM_ストレステスト

## この単元の芯

リスク管理、ALM、ストレステストは、保険会社がどのように壊れるかを先に見に行く単元である。損益計算書で黒字でも、会社が安全とは限らない。生命保険会社は長い負債を持ち、資産は市場で値動きする。金利、株価、為替、信用、解約、死亡率、流動性の変化は、別々に見えても、最終的には支払能力と自己資本へ集まってくる。

ALMは、資産運用のテクニックではない。負債の性質に合わせて資産を持ち、資産と負債のずれから生じる損傷を管理する考え方である。たとえば、遠い将来まで固定利率の給付を約束しているのに、資産が短期中心なら、金利が下がったときに負債価値の増え方へ資産が追いつかない。これが、単なる運用失敗ではなく、資産負債の構造の問題である。

ストレステストは、悪い数字を作る作業ではない。通常のモデルや平均的なシナリオでは見えにくい壊れ方を、経営が意思決定できる形で出す作業である。IAISの保険資本規制やERMの議論でも、リスクを測るだけでなく、資本、リスク許容度、経営行動へ結びつけることが重視される。試験の答案で、リスクの特定、計測、対応、モニタリング、報告が並ぶのは、測定だけでは会社は守れないからである。

ALMの研究でも、資産デュレーションと負債デュレーションを近づけつつ、収益目標や現実的な制約を満たすようポートフォリオを調整する発想が出てくる。試験答案で「資産負債の対応」「金利変動」「解約や新契約を含む将来キャッシュフロー」が出るのは、机上の金利感応度だけではなく、保険会社の実際のバランスシートが時間とともに変わるからである。

## 暗記事項は何を言っているか

「リスク管理プロセス」は、リスクを見つけ、測り、対応し、監視し、報告する流れとして覚える。列挙だけを覚えると弱い。会社がまずやるべきことは、どのリスクを取ってよいかを決めることだ。リスクアペタイト、リスクトレランス、リスクリミットが出るのは、測定結果に意味を持たせるためである。数字が大きいか小さいかだけではなく、自社が許容した範囲なのかを判断する必要がある。

「VaRの限界」は、分布の尻尾や前提の崩れを見落としやすい点にある。VaRは、一定の信頼水準でどれだけ損をするかを見る道具である。しかし、極端な事象の中で何が同時に起きるか、流動性が消えるとどうなるか、モデル前提が崩れるとどうなるかは、VaRだけでは見えにくい。だからストレステストが必要になる。

「流動性リスク」は、資産が足りない話と少し違う。帳簿上は資産超過でも、必要な時点で現金化できなければ支払いに詰まる。大量解約、担保差入れ、再保険回収の遅れ、市場の流動性低下が重なると、資本はあるのに資金繰りが厳しくなる。

## 具体的に見る

長期の円建終身保険を大量に持つ会社を考える。金利が下がると、将来の保険金支払の現在価値は大きく増える。資産側が短い債券中心なら、資産価格も上がるが、負債ほどは増えない。結果として経済価値ベースの純資産が傷む。これは「金利が下がって債券価格が上がるから良い」という単純な話ではない。資産と負債のデュレーションが違うから、同じ金利変化でも効き方が違う。

また、ストレスシナリオでは、金利低下だけでなく、解約率上昇、信用スプレッド拡大、株価下落、再保険回収遅延などが同時に起きる場合を考える。単独では耐えられる変化でも、同時に起きると経営判断が変わる。試験で「ストレステストの意義・目的」が聞かれるのは、通常のリスク量計測だけでは、この同時発生の判断が弱いからである。

## 読み方のコツ

この単元では、用語を次の順に置くと崩れにくい。

1. 会社はどのリスクを取って収益を得ているのか。
2. そのリスクはどの経路で資産、負債、自己資本へ効くのか。
3. 通常モデルで見える部分と、ストレスでないと見えない部分は何か。
4. 測った結果を、資本政策、商品政策、ALM、再保険、配当へどう戻すのか。

「測る」だけでは答案が薄い。測ったあと、経営が何を変えるのかまで書くと、リスク管理の答案になる。
