# リスク管理

## この資料の狙い

リスク管理の章は、用語が横に広がりやすい。  
ERM、ALM、ストレステスト、リスク許容度、経済資本、ソルベンシー、リスクとソルベンシーの自己評価といった話が並ぶので、単語を追うだけだと、全部それっぽく見えてしまう。

この資料では、まず「生命保険会社は何にさらされていて、それをなぜ統合的に見なければならないのか」から入る。  
そのうえで、ALM、ストレステスト、経済価値ベースの考え方、経営での使い方まで、一つの流れとして整理する。

## 教科書との対応

教科書の第4章は、大きく二部構成になっている。

第一部は ERM で、

1. ERM の予備知識
2. ERM の概念
3. ERM の体制
4. リスク管理プロセス

第二部は ALM で、

1. ALM の基本的考え方
2. ALM の留意点

という流れで並んでいる。  
前回の版は ALM とストレステストを中心に説明していて、ERM 側の教科書構成が少し見えにくかった。今回はそこを明示して補う。

## 教科書 第4章を順に読む

第4章は、用語集ではない。  
最初に `何をリスクと呼ぶのか` をそろえ、その次に `それを会社全体でどう受け止めるか` を考え、最後に `ALM のような具体的な管理へどう落とすか` まで進んでいく章である。

なので、前半の ERM と後半の ALM を別の章のように読むと少しもったいない。  
本当は、前半で「会社全体で何を守りたいか」を決め、その考え方を後半で ALM やストレステストの形に落としている。以下では、その流れが自然につながるように、教科書の節ごとに少し丁寧に読みほどいていく。

### 第一部 1.1 リスクと不確実性

最初の節で教科書がやっているのは、言葉の土台づくりである。  
ここを曖昧にしたまま先へ進むと、あとで `リスク量` `リスク資本` `リスク許容度` といった言葉が全部ふわっとしてしまう。

不確実性というのは、将来がまだ一つに決まっていないことそのものを指す。  
金利が上がるか下がるか、株価がどう動くか、死亡率がどう変わるか、解約が増えるか減るか、システム障害が起きるかどうか。こうした「まだ確定していない」という状態をまとめて不確実性と呼ぶ。

その全部が、直ちに経営上のリスクになるわけではない。  
会社として困る形に落ちてきたとき、たとえば損失、資本の毀損、流動性不足、将来利益の悪化、業務継続の支障という形で目的達成を邪魔するときに、はじめて `管理すべきリスク` になる。  
言い換えると、リスクとは、単なる「分からなさ」ではなく、「分からない結果として会社が傷みうるもの」を、管理対象として切り出した概念である。

ここでひとつ大事なのは、アクチュアリーにとってのリスクは、必ずしも期待値のぶれだけではないということだ。  
平均的には問題がなくても、テールで大きく傷むならそれはリスクであるし、モデル化しにくい不確実性でも、経営を壊しうるなら無視できない。教科書が冒頭でこの整理を置くのは、後で出てくるモデルリスクやストレステストの話へつなげるためでもある。

第I部では、ここから `リスクと不確実性の違い` や `生命保険会社における代表的な不確実性` を説明させる形が十分ありうる。暗記というより、土台の定義を言えるようにしておきたい。

### 第一部 1.2 リスクと資本

次の節では、教科書が一気に経営の話へ寄せてくる。  
リスクを知るだけでは足りず、どのくらいの損失なら会社は耐えられるのか、という資本の視点が必要だと言っているのである。

生命保険会社は、短期で店じまいできる業態ではない。  
いま販売している契約だけでなく、過去に引き受けた長い負債を背負っている。だから損失が出たときに大事なのは、単年度利益が赤字かどうかより、予想外の悪化を吸収して約束を守れるだけのクッションがあるかどうかである。そこを表すのが資本である。

ここでいう資本は、会計上の純資産残高をぼんやり眺めることとは少し違う。  
どのリスクがどれだけサープラスを振らすのか、複数リスクが重なったときにどこまで耐えられるのか、資本を食う商品や戦略をどう見直すのか、という意思決定まで含めてはじめて意味がある。  
この意味で、リスク管理と資本管理は別々の仕事ではなく、かなり深くつながっている。

教科書がここを早い段階で置くのは、その後の ORSA、経済価値ベースのソルベンシー規制、ALM、ストレステストが、全部 `最終的にどれだけ余力が残るか` を見たい話だからである。  
第I部では、`リスク資本を考える必要性` や `保険会社で資本管理が重要な理由` がそのまま問われてもおかしくない。

### 第一部 1.3 リスクモデリングの限界

この節は、教科書の中でもかなり大事で、しかも行間が多い。  
モデルを作ると、つい「数字が出たから分かった気になる」。でも教科書は、そこに強くブレーキをかけている。

モデルは、前提、データ、パラメータ、相関、契約者行動仮定、外挿方法の上に立っている。  
だから、何か一つでも土台がずれると、出てくる数字も静かにずれていく。怖いのは、平常時にはそれが目立ちにくいことである。相関が安定している時期、流動性が豊富な時期、契約者行動が過去平均に近い時期は、モデルがかなりうまく見える。

ところが、本当に知りたい危機局面では、そこが壊れやすい。  
市場急変時には相関が一気に高まり、売りたい資産が売れず、解約率が普段と違う動きをし、ヘッジも思ったほど効かないことがある。つまり、モデルは「平時の地図」としては有用でも、「危機の現場」を完全には描き切れない。

アクチュアリーっぽく言えば、ここには少なくとも三種類の限界がある。  
一つ目は、確率分布そのものを取り違えるリスク。二つ目は、パラメータ推定誤差やデータ不足によるリスク。三つ目は、そもそもモデルの外にある制度変更、行動変化、評判悪化のような非モデル化リスクである。  
だから教科書は、モデルを捨てろとは言わず、モデルの外側を見る補助手段としてストレステストや定性的点検を求めている。

第I部では、`リスクモデリングの限界`、`モデル利用時になぜストレステストが必要か` といった形で、かなり素直に出せる節である。

### 第一部 1.4 リスクのフローとストック・ポートフォリオ効果

この節は、一見すると寄り道に見える。  
でも実際には、生命保険会社の数字の読み違いを防ぐためのかなり大事な節である。

フローは、ある期間にどれだけ入ってきて、どれだけ出ていったかを見る発想である。  
保険料収入、保険金支払、新契約件数、解約件数、当期利益などがこちらに当たる。これに対してストックは、ある時点でいま何をどれだけ抱えているかを見る発想で、責任準備金残高、保有資産、含み損益、自己資本、流動性資産残高などが代表例である。

生命保険会社では、この二つを分けて考えないと感覚が狂いやすい。  
たとえば今年の利益フローが良くても、既存の負債ストックが長くて金利感応度が大きければ、将来の経済価値はかなり不安定かもしれない。逆に、当期利益が一時的に悪くても、新契約の質や既存契約群のストックが強ければ、長期的な収益力はそこまで傷んでいないこともある。

ポートフォリオ効果も同じで、個別リスクを一つずつ見ているだけでは会社全体の姿にならない。  
市場リスクと保険引受リスクが完全に同じ向きに動くとは限らないので、統合したリスク量は単純合計より小さくなりうる。ただし、危機時には相関が変わり、普段効いていた分散効果が急に薄くなることもある。  
教科書はここで、「全体を見る」「ただし都合よく分散を信じすぎない」という、後の統合リスク管理の感覚を先に入れている。

第I部では、`フローとストックの違い` や `ポートフォリオ効果をどう理解すべきか` が未出問題の形でも出しやすい。

### 第一部 1.5 リスクの分類と定義の多様性

リスク分類は、正解を一個覚える話ではない。  
ここで教科書が伝えたいのは、分類は管理目的のための道具だということである。

市場リスク、信用リスク、保険引受リスク、流動性リスク、オペレーショナルリスク、事務リスク、戦略リスク、評判リスク。  
こうした分類名はよく出てくるが、どこまでを一つの箱に入れ、どこから別に切るかは、管理したい対象によって変わる。  
たとえば ALM の文脈では金利リスクと流動性リスクの切り分けが重いし、ERM では戦略リスクや評判リスクまで含めて全社で見ないと足りない。会計や規制の文脈では、また別の切り方が前に出てくる。

つまり、分類は暗記リストではなく、「どの単位で見れば経営判断に役立つか」を決める整理である。  
分類の粒度を粗くしすぎると原因が見えなくなり、細かくしすぎると経営の言葉として使えなくなる。教科書は、そのちょうどよい切り方を考える感覚をここで置いている。

第I部では、`リスク分類が一義的でない理由` や `保険会社の主要リスクの例示` が自然な出題になる。

### 第一部 1.6 頻出用語

この節は、単語帳ではなく、リスク管理の会話を成立させるための共通言語づくりである。  
似た言葉が多いので、ここが曖昧だと後ろの章が全部曖昧になる。

たとえば、リスクアペタイトは、会社として「このくらいのリスクは取ってよい」と考える基本姿勢に近い。  
リスク許容度は、その姿勢をもう少し管理可能な形にしたもので、「どこまでのぶれなら我慢するか」という境界線に近い。リスクリミットはさらに現場寄りで、日々の運営で超えてはいけない具体的な上限である。  
KRI は、問題が表面化する前に異変の兆しを捉えるための指標で、結果指標というより前触れを見るための計器だと理解するとよい。

他にも、エマージングリスクは、まだ十分には見えていないが将来大きな影響を持ちうるリスクであり、モデルリスクは、モデルを使うこと自体に伴うリスクである。  
ベーシスリスクは、ヘッジ対象とヘッジ手段が完全には一致しないことで残るズレ、カウンターパーティーリスクは、相手先が約束どおり履行しないことで生じるリスクである。  
言葉だけだと似て見えるが、誰が、どの場面で、何のために使う言葉かまで押さえると一気に整理しやすくなる。

第I部では、ここをそのまま `違いを説明せよ` と問う形がかなり作りやすい。

### 第一部 2. ERM の概念

ERM は、全部のリスクを小さくする仕組みではない。  
それだと会社は何もできなくなってしまう。そうではなく、会社の目的に照らして、どのリスクを、どの程度、どの条件で引き受けるかを全社で整える考え方である。

ここで大事なのは、ERM が `リスク回避の技術` ではなく `経営そのもののフレーム` だということである。  
商品をどう売るか、資産をどう持つか、どこまで再保険を使うか、どのくらいの資本を置くか、どこまで配当や株主還元を行うか。これらは全部、収益機会とリスクを同時に見る話である。  
個別部門が自分の担当リスクだけを最適化すると、会社全体ではちぐはぐになることがある。ERM は、それを避けるための「全体の座標軸」だと思うとよい。

生命保険会社で特にこれが重いのは、資産と負債、商品と運用、販売と資本、健全性と収益性が強くつながっているからである。  
教科書がこの節を独立で置くのは、ALM も ORSA もストレステストも、全部この統合的な考え方の具体化にすぎないからである。

第I部では、`ERM の概念` や `なぜ生命保険会社で統合的リスク管理が必要か` が定番の聞かれ方になる。

### 第一部 3. ERM の体制

考え方だけでは、ERM は回らない。  
この節で教科書が見ているのは、理念を会社の意思決定へどう埋め込むかである。

まず、取締役会や経営陣が、どのリスクをどこまで取るかについて責任を持たなければならない。  
そのうえで、第一線で実際にリスクを取る業務部門、第二線でルールを整え独立に牽制するリスク管理部門、第三線で仕組みが機能しているかを検証する内部監査が、それぞれ別の役割で噛み合う必要がある。いわゆる 3 線管理は、この役割分担を整理したものだと読めばよい。

COSO をここで学ぶ意味も同じである。  
内部統制のフレームを通じて、リスク管理が一部門の仕事ではなく、ガバナンス、方針、情報伝達、モニタリングまで含んだ組織活動だと理解させてくれる。  
また、ORSA は単なる年次報告書ではなく、会社が自分のリスク特性、資本十分性、将来シナリオを前向きに点検して、経営計画へ戻す仕組みである。ここまでつながって初めて、ERM 体制が「ある」ではなく「回っている」と言える。

第I部では、`取締役会・業務部門・リスク管理部門・内部監査の役割`、`ORSA の意義`、`3 線管理` がそのまま問われやすい。

### 第一部 4. リスク管理プロセス

この節は、ERM の考え方を日々の運営へ落とした姿である。  
リスク特定、評価、計測、対応、モニタリング、報告という流れは、単に順番を覚えるために並んでいるのではない。

まず、何がリスクなのかを特定しないと、測るべきものが決まらない。  
次に、それがどの程度大きく、どう広がり、どんな条件で悪化するかを評価する。そこでは定量モデルが使えることもあれば、定性的な判断が中心になることもある。  
さらに、対応として回避、低減、移転、保有のどれを選ぶかを決める。最後に、結果を継続的に見張って、必要なら経営へ戻す。この一連の循環が回ってはじめて、リスク管理は `点検` ではなく `経営管理` になる。

アクチュアリーの感覚で補うなら、計測尺度の選び方にも意味がある。  
VaR なのか TVaR なのか、ファクターベースで見るのか、シナリオで見るのか、確率論的に見るのかで、見えるものが違う。  
リスク集約でも、単純合算は保守的だが荒いし、分散共分散法は扱いやすいが正規性や線形近似への依存がある。教科書は細かな数式に深入りしなくても、「測る方法を選ぶこと自体が経営判断である」という感覚を持たせようとしている。

第I部では、`リスク管理プロセスの各段階`、`リスク対応の類型`、`モニタリングと報告の役割`、`リスク集約方法の考え方` が出題しやすい。

### 第二部 1. ALM の基本的考え方

後半に入ると、統合リスク管理の考え方が、いよいよ資産と負債の具体的な管理へ落ちてくる。  
ALM は、資産だけを上手に運用する話でも、負債だけを丁寧に評価する話でもない。資産と負債を一緒に見て、その差額が環境変化で壊れにくいようにする管理である。

なぜそこまで資産と負債を一緒に見る必要があるのか。  
生命保険会社の負債は長く、しかも金利や解約行動に反応する。資産の利回りだけが高くても、負債価値の増え方のほうが大きければ、サープラスは悪化する。  
だから ALM では、表面利回りよりも、キャッシュフローの時期、現在価値、デュレーション、コンベクシティ、流動性、契約者オプションといった、資産と負債を同じ物差しで比べられる要素が前に出る。

もちろん、現実には完全マッチは難しい。  
市場に十分な超長期資産がない、取引コストがある、商品に最低保証や解約オプションが付いている、といった制約があるからである。だから教科書は、ALM を一回合わせて終わる技術ではなく、戦略を立て、実行し、ずれを見て、修正し続けるプロセスとして描いている。

第I部では、`ALM の意義`、`デュレーションやコンベクシティを見る理由`、`ALM が継続的管理である理由` が基本論点になる。

### 第二部 2. ALM の留意点

最後の節は、理論を実務へ持ち込んだときに何が難しくなるかをまとめる場所である。  
ここまで来ると、ALM は単なる金利管理ではなく、商品設計、契約者行動、流動性、ヘッジコスト、会計処理、規制対応まで巻き込む話になる。

たとえば、流動性リスクは、資産と負債の現在価値が合っているだけでは防げない。  
大きな解約や保険金支払いが短期に集中すると、理論上は健全でも、現金化のために不利な価格で売却せざるをえず、損失が実現することがある。  
また、変額年金やユニット・リンク商品では、最低保証が実質的にオプションとして働くので、株価、金利、ボラティリティ、時間の変化が負債価値へ効いてくる。MVA や利率変動型商品の設計は、こうしたズレを契約条件の側から和らげる工夫として理解できる。

要するに、この節が言いたいのは、ALM の難しさは商品が変わると中身も変わるということである。  
一時払終身では超長期金利資産の確保が重くなり、変額年金では保証対応が前に出て、就業不能や介護では給付発生パターン自体の不確実性が強くなる。  
だから ALM の留意点は、一般論だけで終わらず、商品別・会社別の文脈へ落として考えなければならない。ストレステストやリバース・ストレステストが重要なのも、その会社特有の弱点をあぶり出して経営へ返すためである。

第I部では、`流動性リスクの意味`、`商品特性で ALM の難しさが変わる理由`、`MVA や利率変動型商品の位置づけ`、`ストレステストとリバース・ストレステストの意義` が未出含めて狙われやすい。

## まず、この章は何を解決したいのか

生命保険会社は、将来の約束を大量に抱えている。  
しかもその約束は、金利、株価、為替、死亡率、解約率、事業費、新契約動向、大災害、システム障害など、いろいろな要因で影響を受ける。

ここで難しいのは、リスクが一つずつ別々に起きるとは限らないことだ。  
金利が動けば負債価値も資産価値も変わるし、景気悪化は解約動向や資金繰りにも波及しうる。大地震は保険金支払いだけでなく、市場や業務継続にも効く。つまり、リスクはつながって動く。

リスク管理の章がやろうとしているのは、この複雑さを放置せず、

- どんなリスクを抱えているかを特定し
- それをどう測り
- どこまで引き受けるかを決め
- 超えてはいけない線を置き
- 問題が出たときの対応を準備し
- その結果を経営判断に戻す

という一連の管理の型を作ることである。

```mermaid
    B --> C["リスク許容度・リスク戦略"]
    C --> D["対応"]
    D --> E["モニタリング・報告"]
    E --> F["経営判断"]
    E --> A
    G["市場・保険引受・流動性・オペ・評判"] --> A
```

この章は、リスクの種類を並べる章ではなく、`会社がどのリスクを、どこまで引き受け、それを経営へどう返すか` を一周で見る章である。
だから ERM、ALM、ストレステスト、リスク許容度は、別論点ではなく同じ輪の別の位置にある。

## リスクと不確実性は何が違うのか

ここは地味だが、章全体の入口になる。  
不確実性は、将来が確定していないこと全般を指す。これに対してリスクは、その不確実性のうち、会社の目的達成に影響するものを、ある程度特定し、評価し、管理対象として扱うときの言葉だと考えると分かりやすい。

つまり、世の中には不確実なことが無数にあるが、その全部を同じ熱量で追うわけにはいかない。  
会社の支払能力、収益、資本、評判、流動性、業務継続に響くものを選び出し、管理可能な形に変える。その変換作業がリスク管理の出発点である。

## リスク資本を考えるのはなぜか

教科書の ERM 予備知識には、リスク資本の話も置かれている。  
これは、「どんなリスクがあるか」を知るだけでは足りず、そのリスクに対してどのくらいの資本的な余力が必要かを考えなければ、経営判断に戻せないからである。

保険会社は、損失が出てもすぐに契約を止められない。  
長期契約を抱えたまま耐えなければならないので、どこまでの損失に耐える設計なのか、どこで資本政策やリスク抑制策が必要になるのかを、資本の観点から見る必要がある。

つまりリスク資本は、リスク管理を `注意喚起` で終わらせず、経営資源配分の話に変えるための橋渡しである。

## 必要資本要件の水準は、何をインプットに決めるのか

ここは `必要資本は数式だけで自動的に一意に決まるわけではない` と押さえる論点である。
監督上どこまで耐えるべきか、格付をどう置きたいか、利益のぶれをどこまで許容するか、といった経営判断が資本水準に入ってくる。

だから答案では、少なくとも次の6つを `必要資本要件の主なインプット` として切れるようにしておきたい。

| インプット | 何を見ているか | 生命保険会社での意味 |
| --- | --- | --- |
| 監督当局が要請する資本要件 | 規制上の最低線 | ここを割ると早期是正措置や信認低下の問題になる |
| 望ましい格付 | 外部から見た支払能力の水準 | 再保険条件、資金調達、販売信認に響く |
| 望ましい収益のボラティリティ | 利益変動をどこまで許すか | 単年度損益のぶれと健全性の両立を考える |
| 望ましい株主リターン | 資本効率との両立 | 厚く持てば安心だが、資本効率は下がりやすい |
| リスクの蓄積 | 既に抱えているポジションの重さ | 高予定利率契約や保証商品の蓄積が厚いほど重くなる |
| 市場の期待 | 投資家・取引先・販売市場の見方 | 規制充足だけでは足りず、「安心して任せられるか」が問われる |

たとえば、規制上は足りていても、高格付を維持したい会社や、保証性商品の比率が高い会社では、社内で置く必要資本はもっと厚くなることがある。
逆に、資本効率だけを優先して薄く置くと、平時は見栄えがよくても、格下げや販売不振を通じて後で高くつく。
この論点は、`必要資本は純粋な数理計算ではなく、戦略と制約を背負った数字である` と理解すると崩れにくい。

## なぜ統合的に見なければならないのか

生命保険会社では、リスクを縦割りで見ると、見落としが出やすい。  
たとえば市場リスクだけ見て資産側を守っていても、負債側の金利感応度がもっと大きければ、会社全体のサープラス、つまり資産と負債の差額は不安定なままである。保険引受リスクだけ見ても、解約急増時の流動性リスクや評判リスクを外すと、危機の入り口を見逃す。

統合リスク管理が必要なのは、会社が守りたいものが最終的には一つだからである。  
それは「長期にわたって約束を履行できること」であり、そのために必要な資本、流動性、収益性、業務継続性をまとめて見ないといけない。

金融庁の監督指針でも、リスク管理方針、リスク許容度、ストレステスト、リスクとソルベンシーの自己評価、資産負債の総合的な管理が一続きで並んでいる。  
これは、管理が分業であっても、意思決定は分断してはいけないということを示している。

### ミニ例: 金利低下と解約増加が同時に来るとき

| 起きたこと | 資産側で起こること | 負債側で起こること | 経営上の問題 |
| --- | --- | --- | --- |
| 金利低下 | 再投資利回りが低下しやすい | 超長期負債の現在価値が膨らみやすい | サープラスが縮む |
| 解約増加 | 資産を早めに売る必要が出る | 負債キャッシュフローが前倒しになる | 流動性が苦しくなる |

もし金利低下だけを市場リスクとして見ていると、解約急増による資金繰り悪化を外しやすい。
逆に流動性だけ見ていると、負債価値の膨張を見落とす。統合的に見るとは、こういう `別のリスクが同時に会社を押す場面` を前提に置くことである。

## ERM の予備知識として何を押さえるべきか

教科書の最初の節には、リスクと不確実性だけでなく、リスク資本、リスクモデリングの限界、リスクカテゴリーの多様性、頻出用語などが並ぶ。  
これは少し散らばって見えるが、全部「数字を出す前に土台をそろえる」ための材料である。

リスク資本の話は、会社がどのくらいの損失まで耐えたいのかを考える入口になる。  
モデリングの限界の話は、きれいな数字が出ても、それで現実を取りこぼすことがあると教えてくれる。リスク分類の話は、どのリスクをまとめて見て、どこから別物として扱うかの整理である。

つまりこの節は、個別論点の寄せ集めではない。  
 ERM を始める前に、「何をリスクとして扱うのか」「数字は何を表していて、何を表しきれないのか」をそろえるための準備運動である。

## リスクモデリングの限界を先に学ぶのはなぜか

教科書本文を読むと、この章はかなり早い段階でモデルの限界に触れている。
これは健全で、モデルを使う章ほど、先に限界を知っておかないと危ないからである。

モデルは、過去データ、仮定、パラメータ、相関の置き方に依存する。
しかも、まさに危ない局面ほど、過去の経験が役に立ちにくく、相関が壊れ、流動性が消え、行動変化が起こりやすい。つまり、知りたいときほど外しやすい。

だから教科書は、モデルを否定するのではなく、`モデルを使うなら、モデルの外側も見よ` と言っている。
これが後のストレステストや定性的評価の必要性につながる。

答案で切るなら、モデルが危うくなる典型場面は次のように整理するとよい。

| 外れやすい場面 | 何が起きているか | 典型的な対応 |
| --- | --- | --- |
| データが現在の状況を代表していない | 過去データでは今の市場や商品構成を表せない | 前提見直し、外部データ、専門判断で補う |
| 暗黙の前提が壊れている | 正規性、相関安定、行動一定などが崩れる | ストレステスト、感応度分析で補う |
| 顧客行動が変わる | 解約率、更新率、オプション行使が急変する | 行動シナリオを別立てで置く |
| テールへ無理に外挿している | 観測していない極端事象をモデルが言い切ってしまう | シナリオ分析、リバース・ストレステストを使う |
| 数字だけが一人歩きする | 利用者がモデルの限界を忘れてしまう | モデルガバナンス、第三者レビュー、限界の明示 |

とくに危ないのは、モデルが外れること自体より、`数字が出たから安心だ` と経営が思ってしまうことである。
2023-2 の論点は、モデル批判ではなく `モデルを使う側のリスク管理` を問うている。

## ERM の概念と体制をなぜ分けて学ぶのか

ERM の概念は、「会社全体の目的に照らして、リスクを統合的に管理する」という思想の話である。  
これに対して体制は、その思想を誰が、どの場で、どんな役割分担で回すかの話である。

この2つを分ける意味は大きい。  
思想だけ立派でも、取締役会、経営陣、リスク管理部門、業務部門、内部監査の役割が曖昧なら回らない。逆に、会議体や部署が揃っていても、何のための管理かが曖昧なら形骸化しやすい。

教科書が `ERM の組織的要素とその評価` と `リスク戦略` を並べているのは、その両面を見ているからである。  
組織図だけ覚えるのではなく、経営戦略とリスク戦略がどうつながるかまで見る必要がある。

## リスク戦略は経営戦略の言い換えではない

ここも教科書の大事なところで、リスク戦略は単に「慎重にやります」という宣言ではない。  
会社がどんな利益機会を取りにいき、そのためにどんなリスクをどの順序で引き受け、どこに線を引くかを具体化したものがリスク戦略である。

たとえば、新商品を伸ばしたい、外貨建資産を増やしたい、保証性商品を強めたいという経営戦略があれば、その裏には当然リスクの取り方がある。  
リスク戦略は、その裏面を明示するものだと考えると分かりやすい。

## COSO や 3 線管理を学ぶ意味

教科書と単元別ノートには、COSO や 3 lines of defense の話も出てくる。
これを単なる用語暗記で終わらせるのは惜しくて、ここで見ているのは「誰が何を抑えるのか」という統制の設計である。

業務部門がリスクを取り、リスク管理部門が横から見る。
内部監査がさらに独立して点検する。このように役割を分けるのは、リスクを取る人が同時に自分を採点すると甘くなりやすいからである。

つまり COSO も 3 線管理も、管理の正しさそのものより、管理が思い込みで暴走しない仕組みをどう作るかの話である。

たとえば外貨建商品の販売を強めたい場面を考えると分かりやすい。
営業現場は売上を伸ばしたいので、どうしても商品魅力や販売機会に目が向きやすい。そこへリスク管理部門が、為替感応度、ヘッジコスト、解約時の流動性を横から見る。さらに内部監査が、「その管理が本当に回っているか」を後ろから点検する。
COSO の 5 要素も、結局はこの仕組みを崩さないためにある。統制環境で土台を作り、リスク評価で何が危ないかを洗い、統制活動で抑え、情報と伝達で回し、モニタリングで緩みを見つける、という順で読むと意味が通りやすい。

ここは、答案としては次の2枚をそのまま切れると強い。

### 3つの防衛ライン

| ライン | 主体 | 役割 | 外貨建商品販売の例 |
| --- | --- | --- | --- |
| 第1ライン | 業務部門・現場 | リスクを取りつつ日々管理する | 営業・商品・運用が販売、引受、ヘッジを実行する |
| 第2ライン | リスク管理・コンプラ等 | 横断的ルール設定、モニタリング、けん制 | 為替感応度、ヘッジ方針、流動性、限度超過を点検する |
| 第3ライン | 内部監査 | 第1・第2ラインが本当に機能しているか独立評価する | 管理ルールが実際に回っていたかを検証する |

大事なのは、第2ラインが `何でも止める部署` でも `現場の下請け` でもないことである。
第1ラインに異議を申し立てられる独立性が必要だが、同時に、情報が上がってくる信頼関係も必要になる。

### COSO の5要素

| 要素 | 何を担うか |
| --- | --- |
| 統制環境 | 組織文化、責任分担、規律の土台 |
| リスク評価 | 何が危ないかを見つけ、優先順位をつける |
| 統制活動 | 承認、牽制、ルール、チェックなどの具体策 |
| 情報・伝達 | 現場の異変や判断材料を必要な相手へ届かせる |
| 監視活動 | 仕組みが緩んでいないか継続的に点検する |

内部統制は、不正防止だけの話ではない。
`業務の有効性・効率性` `財務報告の信頼性` `法令遵守` に関する合理的保証を与えるためのプロセスだと切れると、COSO が ERM の体制論に出てくる意味が見えやすい。

## リスク管理プロセスは何を回しているのか

リスク管理プロセスは、ふつう

1. リスク特定
2. リスクアセスメント
3. リスク計測
4. リスク対応
5. リスクモニタリング
6. リスク報告

の流れで説明される。
この順番を丸暗記するだけでは弱くて、本当に見たいのは「会社が何を回しているのか」である。

リスク特定では、どんな事故が起きるかを数えるのではなく、会社の目的を壊しうるものを拾う。
アセスメントでは、その重要度や性質を見極める。計測では、できるものは数字にする。対応では、回避、低減、移転、保有などの方針を決める。モニタリングでは前提が崩れていないかを見る。報告では、経営が動ける形へ翻訳する。

要するに、リスク管理プロセスは `測る流れ` ではなく `意思決定を回す流れ` である。

答案で6段階をそのまま切るなら、次のように並べると崩れにくい。

| 段階 | ここでやること | ここが抜けると何が起きるか |
| --- | --- | --- |
| リスク特定 | 市場、保険、流動性、戦略、評判まで洗い出す | そもそも見ていないリスクは管理できない |
| リスクアセスメント | 重要度、発生可能性、影響度を見極める | 優先順位が付かず、論点が散る |
| リスク計測 | 必要なものを数字に落とす | 限度設定や資本配賦に使えない |
| リスク対応 | 回避、受容、軽減、移転（共有）を選ぶ | `測っただけ` で終わる |
| リスクモニタリング | 前提崩れ、KRI、限度超過を見張る | 環境変化に気づけない |
| リスク報告 | 経営が動ける形へ翻訳して返す | 意思決定につながらない |

たとえば、変額年金の保証リスクが重すぎるなら、販売停止は回避、保証水準見直しは軽減、再保険は移転、十分な資本を置いて継続するなら受容である。
同じリスクでも、会社の戦略と資本余力で選ぶ対応が変わる。ここが `経営判断` としてのリスク管理である。

## リスク対応にはどんな型があるのか

教科書本文では、リスク対応のところで、回避、低減、移転・共有、受容のような考え方が見える。  
ここは実務上かなり大事で、リスクを見つけたからといって、全部減らすしかないわけではない。

たとえば、ある商品を売らないというのは回避である。  
ヘッジや商品設計見直しで感応度を下げるのは低減である。再保険や ART で第三者へ渡すのは移転・共有である。一定範囲なら自社で抱えるのは受容である。

どれを選ぶかは、リスクの大きさだけでなく、収益性、コスト、実行可能性、会社の戦略との整合で決まる。  
だからリスク対応は、管理技術というより、経営の選択そのものである。

## リスク報告はなぜ最後に置かれるのか

教科書のプロセスが最後にリスク報告で終わるのも意味がある。  
報告は単なる共有ではなく、ここまでの分析を経営が動ける言葉へ変える段階だからである。

数字が精密でも、取締役会や経営陣が `何が危ないのか` `何を決めるべきか` を読めなければ管理は動かない。  
だから報告は、最後の事務ではなく、リスク管理が経営へ接続される出口と考えた方がよい。

## リスクとソルベンシーの自己評価を独立で置く意味

監督指針でいうリスクとソルベンシーの自己評価は、単なる年次報告ではない。  
会社が自分のリスク特性と将来の資本十分性を、自分の言葉で見直す場である。

ここで重要なのは、現在だけでなく中長期を見ることだ。  
新規事業計画、商品設計、販売見通し、経済環境の変化を踏まえて、「このまま進んだとき本当に持つのか」を考える。だから自己評価は、過去の点検よりも、前向きの経営計画に近い面を持つ。

教科書本文でも、フォワードルッキングな評価や内部モデルの話が出てくるが、その重心はここにある。  
自社の将来を、自社のリスクプロファイルに即して見よ、ということである。

たとえば、これから 3 年で外貨建一時払商品を大きく伸ばす計画がある会社なら、自己評価では「今の資本で足りているか」だけでは足りない。  
販売増に伴って、為替リスク、解約集中時の流動性リスク、ヘッジ調達コスト、販売チャネル依存がどう変わるかを先に見にいく必要がある。今は健全に見えても、計画を乗せると 2 年後に耐えにくくなるなら、その時点で販売政策やヘッジ方針、資本計画を修正すべきだからである。  
自己評価は、現状説明の報告書というより、「このまま走ったらどこで苦しくなるか」を先回りして確かめる経営ツールとして読むのがよい。

## 監督規制上の資本と経済資本は、なぜ同じ数字にならないのか

どちらも `必要な資本` を見ているようで、目的が違う。
監督規制は、まず全社に共通の最低線を置いて、客観性と比較可能性を確保したい。これに対して経済資本は、会社固有のリスクプロファイルに照らして、どこまで本当に耐えられるかを見たい。

そのため、両者の差は次のような形で現れやすい。

| ずれやすい点 | 監督規制上の資本 | 経済資本 |
| --- | --- | --- |
| 主目的 | 比較可能性、実行可能性、最低線の監督 | 個社管理、資本配賦、リスク・リターン判断 |
| ボラティリティの見方 | 標準化された前提になりやすい | 自社実績や内部モデルを反映しやすい |
| 分散効果 | 保守的・定型的に扱われやすい | 相関やコピュラ等で個社の依存関係を見やすい |
| 対象リスク | 共通枠に乗せやすいものが中心 | オペ・評判・戦略なども含めて見やすい |
| 資本として認める範囲 | ルールベース | のれん、税効果、質の評価などで見方が分かれやすい |

たとえば、規制では拾いにくいオペレーショナルリスクや、商品・チャネル特有の解約リスクは、経済資本では重く出ることがある。
逆に、規制では比較可能性のために保守的な係数を置くので、個社実態より重く見える部分もある。
だから `規制で足りているから経済資本でも足りている` とも、`経済資本で足りているから規制も問題ない` とも言い切れない。

## リスク管理は「測ること」で終わらない

この章でありがちな誤解は、モデルで数字を出すことがリスク管理だと思ってしまうことである。  
もちろん測定は重要である。VaR、経済資本、感応度、シナリオ分析など、数字にしないと比べにくいものは多い。

でも、本当に大事なのは、その数字をどう使うかである。  
たとえば、

- 商品をどう設計するか
- 最低保証をどこまで付けるか
- 資産運用方針をどう置くか
- 配当や内部留保をどう調整するか
- 資本政策をどう組むか

に戻らないなら、計測だけ精密でも経営は強くならない。

この章は、測定技法の章であると同時に、経営管理の章でもある。  
だから、第I部でも定義と計算だけでなく、リスク戦略、リスク許容度、ストレステストの活用目的まで問われやすい。

## ALM は何を守ろうとしているのか

ALM は、資産と負債を一緒に見る管理である。  
でも、これを「債券のデュレーションを合わせる技術」とだけ覚えると足りない。

生命保険会社の負債は長い。  
遠い将来の保険金や年金の支払いは、今の金利で割り引いた現在価値として評価すると、金利に敏感に動く。一般に、キャッシュフローの受払時点が遠いほど、現在価値は金利変化の影響を強く受ける。これをざっくり測る指標がデュレーションである。

ここで、負債が超長期なのに、資産が短い債券ばかりだとどうなるか。  
金利が下がると、遠い将来にある負債の現在価値は大きく増えやすい。一方、短い資産は値上がりするにしても増え方が相対的に小さい。すると、資産は増えるが負債はもっと増える、ということが起きる。これがサープラスの悪化である。

たとえば、時価ベースで資産110、負債100、差額10の会社があったとする。  
金利低下後に資産が115、負債が120になれば、差額は 10 からマイナス5へ悪化する。これが「資産と負債の動きがずれる」ときに起きることである。

ALM が守ろうとしているのは、この差額が環境変化で壊れにくい状態である。  
つまり、資産だけをうまく運用する話ではなく、資産と負債の組み合わせとして会社の耐久力を高める話だと考えるとよい。

デュレーションで言い換えると、負債デュレーションが 15、資産デュレーションが 5 という状態では、金利が 1% 下がったとき、ざっくり負債は 15% 程度、資産は 5% 程度動く向きになる。もちろん実務ではコンベクシティやキャッシュフロー形状も入るが、まずは「長いものほど金利で大きく動く」という直感をここでつかむのが大事である。

## ALM は一回合わせて終わりではない

教科書の ALM の基本的考え方を読むと、ALM は継続的な管理として描かれている。  
それは自然で、資産も負債も時間とともに形が変わるからである。

金利が動けば資産価値も負債価値も変わる。  
解約が増えれば負債キャッシュフローの時期が前倒しになる。新契約が増えればポートフォリオ構成が変わる。だから、ある時点でデュレーションを合わせたとしても、そのままではすぐにずれていく。

ALM を `一度きれいに作る技術` ではなく、`ずれ続けるものを見続けて直す管理` として理解することが大事である。

## デュレーションとコンベクシティはどう見るか

デュレーションは、金利が少し動いたときに価格がどのくらい動くかを測る、一次近似の指標である。  
満期までが長いほど、一般にはデュレーションは長くなる。だから超長期の保険負債は、金利変化に敏感になりやすい。

ただ、金利変化が大きいと、一次近似だけでは足りない。  
そこでコンベクシティが要る。これは、金利と価格の関係の曲がり方を表す指標で、デュレーションだけでは拾いきれない部分を補う。

試験ではここを数理理論として深く問うというより、「なぜデュレーション・マッチングだけで終わらず、コンベクシティも見るのか」を理解しているかが大事である。  
ざっくり言えば、少しの金利変化なら傾き合わせで足りても、大きく動くと曲がり方の違いが効いてくるからである。

式の形だけ触れておくと、価格変化率はざっくり `-デュレーション×金利変化 + 1/2×コンベクシティ×(金利変化)^2` で近似される。  
だから、金利がほんの少ししか動かないなら一次項のデュレーションが効きやすいが、変化幅が大きくなるほど二次項のコンベクシティの差が無視しにくくなる。教科書がこの二つを並べるのは、単に指標を増やしたいからではなく、金利変動の大きさに応じて見落としを減らすためである。

## 流動性リスクはなぜ ALM とつながるのか

ALM を金利リスクだけの話にすると不十分である。  
生命保険会社では、解約や資金流出が増えたとき、資産を予定より早く売らなければならないことがある。

このとき問題になるのは、売りたくないタイミングで売ることだ。  
長期で保有していれば回収できる資産でも、急に現金化しようとすると、価格下落局面で売却損が実現することがある。つまり、会計上の評価損だったものが、現実の損失に変わる。

解約増加時の耐久性低下とは、まさにこのことである。  
想定したより早く資金が出ていくと、資産運用の前提が崩れ、ALM が組んでいた時間構造が壊れる。だから ALM では、負債の金利感応度だけでなく、解約オプションや流動性バッファーも見なければならない。

たとえば、10 年持つ前提で買った長期債を、解約急増で 2 年目に大量売却しなければならなくなると、その債券は本来予定していた利回りで持ち切れない。  
しかも売却時が金利上昇局面なら、評価損がそのまま実現損になる。流動性リスクが怖いのは、単に資金が足りないからではなく、「長く持てば問題が小さい資産」を、悪いタイミングで手放させるからである。

## オプションと保証が入ると何が難しくなるのか

変額年金の最低保証や、解約返戻金、MVA、利率変動型商品の見直し条項など、保険契約にはオプション的な要素が入ることがある。  
ここが難しいのは、契約者の行動が市場環境と結びつくからである。

金利が動いたときに、会社の資産価値だけが変わるのではない。  
契約者が解約するか、継続するか、一時金を選ぶか、年金を選ぶかといった行動も変わりうる。そうすると、負債キャッシュフローの形自体が変わる。

だから、保険負債は単なる固定された将来支払いではない。  
保証とオプションを含む負債として見なければならず、ALM やストレステストもその前提で組む必要がある。

## ALM の留意点として教科書が強調していること

教科書の ALM パートでは、準備、資産負債管理の対象となるリスク、さまざまな商品区分への資産負債管理型の適用、資産負債管理の手法、ミスマッチ管理方法が並ぶ。  
これはつまり、ALM をただの理論ではなく、商品特性ごとに運用へ落とすとどう難しくなるかを見ている。

たとえば、終身保険のように負債が超長期のものと、貯蓄性商品のように解約感応度が高いものでは、同じ ALM でも難しさが違う。  
区分経理や資産区分の考え方が入ってくるのも、負債特性の違うものを一つの箱で運用すると、ミスマッチが見えにくくなるからである。

ここで教科書が伝えたいのは、ALM に万能の型があるわけではないということである。  
会社の負債構成、販売商品、顧客行動、使える市場、会計制度、監督規制によって、実際の運営はかなり変わる。

たとえば、一時払終身のように契約初期に大きな資金が入る商品では、比較的素直に長い円金利資産へ置いていく ALM を考えやすい。  
一方、変額年金の最低保証のように市場変動と契約者行動が絡む商品では、単純な債券保有だけでは足りず、保証コストやヘッジの持続可能性まで見る必要が出る。同じ ALM でも、商品が違うと難しさの中身が変わる。

## 商品ごとに ALM の難しさが違うのはなぜか

教科書の ALM パート後半は、商品別に読むとかなり理解しやすい。  
たとえばユニット・リンク商品や変額年金では、外部指数や保証が絡むので、資産側のリンクと保証コストの管理が重くなる。定期保険では、解約返戻金が小さい分だけ単純に見えても、長い保障期間の中でどの資産を対応させるかの判断が要る。

就業不能性商品や介護商品では、保険事故の発生パターン自体が通常の死亡保障と違う。  
つまり、ALM の難しさは金利だけではなく、商品設計そのものに左右される。

だから ALM は、資産運用部門だけの仕事ではない。  
商品開発、保険数理、営業戦略、再保険戦略とつながっている。

## すべてを完全にマッチさせなくてもよいのはなぜか

教科書の ALM 基本思想を読むと、すべての資産と負債を完全に複製することまでは必ずしも求めていない。  
これは現実的で、コスト、市場の厚み、取引可能な年限、商品特性を考えると、完全な一致は難しいからである。

大事なのは、どのミスマッチをどの程度まで許容し、どこを優先して抑えるかを決めることだ。  
つまり ALM は、理論上の完全解を探す競技ではなく、限られた市場とコストの中で会社を壊れにくくする実務である。

## ストレステストは何のためにやるのか

VaR や通常のリスク計測は便利だが、万能ではない。  
過去データからの統計推計に依存するので、本当に嫌な危機局面をそのまま捉えられるとは限らない。過去に十分なデータがない事象、複数リスクの同時変動、相関の崩れ、流動性低下のような局面は、通常のモデルだけだと見落としやすい。

そこでストレステストが要る。  
ストレステストは、「もし本当に困る事態が起きたら会社はどうなるか」を、シナリオで確かめる道具である。

ここで大事なのは、悲観的であればいいわけではないことだ。  
監督指針でも、ヒストリカルシナリオだけでなく仮想シナリオも用い、複数リスクの同時変動や市場流動性低下も含めて検討すること、結果を具体的な判断に活用することが求められている。

つまり、ストレステストは「怖い絵を描いて終わる」ものではない。  
資本、流動性、商品、再保険、ヘッジ、BCP、リスクリミットの見直しにどうつなげるかまで含めて意味がある。

実務で置きやすいのは、歴史シナリオと仮想シナリオの二本立てである。  
前者なら、過去の金利急変や株価急落を引き写して、自社バランスシートに当てる。後者なら、金利低下と解約急増と信用スプレッド拡大が同時に起きる、といった自社に痛い組み合わせを作る。そのうえで、基礎利益、純資産、ソルベンシー指標、資金繰り、流動性資産の残高まで見て、どこで経営対応が必要になるかを確認する。  
大事なのは、数値が悪かったあとに「では販売を抑えるのか、ヘッジを増やすのか、流動性バッファーを積むのか」まで話が進むことである。

## 金利リスクと流動性リスクはどう違うのか

教科書のミスマッチ管理方法では、金利リスクと流動性リスクが分けて論じられている。  
これは重要で、両方とも資産と負債の時間構造のずれに関係するが、困り方が違うからである。

金利リスクは、主に価値のずれとして現れる。  
金利が動くと、資産価値と負債価値の変化幅が違い、サープラスが揺れる。

流動性リスクは、主にタイミングのずれとして現れる。  
今すぐ払う現金が要るのに、手元資産がそのタイミングに合っていないと、損を承知で売るしかなくなる。価値の問題が、その場の資金繰り問題へ変わる。

教科書が両方を分けるのは、同じミスマッチでも、必要な管理手段が少し違うからである。

## 再保険やデリバティブで移した後のリスクも見なければいけない

リスクを第三者へ渡せば終わり、とはならない。  
教科書や監督指針が再保険やデリバティブを語るときに、カウンターパーティーや残余リスクへ触れるのはそのためである。

再保険で保険リスクを移しても、相手が本当に払えるかという信用リスクは残る。  
デリバティブでヘッジしても、流動性やヘッジコスト、相関崩れの問題が残る。つまり、移転は消去ではなく、形を変えることが多い。

この感覚を持っておくと、「リスク移転をしたから安全」という雑な理解を避けやすい。

## リバース・ストレステストは何を見るのか

ふつうのストレステストは、先にシナリオを置いて結果を見る。  
これに対してリバース・ストレステストは、「会社が危うくなる状態」を先に置き、そこに至る筋道を逆算する。

この考え方が有効なのは、見逃していた弱点をあぶり出しやすいからである。  
たとえば、単独の市場ショックでは耐えられても、解約急増と信用スプレッド拡大と資金繰り悪化が重なると危ない、というような複合経路が見えることがある。

経営にとって大事なのは、「何が起きたら終わるのか」を曖昧なままにしないことだ。  
リバース・ストレステストは、その境界線を言語化するための道具でもある。

たとえば、「ESR が一定水準を割り、同時に手元流動性が数か月もたない状態」を危機ラインと置く。  
そのうえで、そこへ至る道筋として、金利低下、株安、解約急増、再保険回収遅延、ヘッジコスト上昇がどの順で重なると危ないかを逆算する。こうすると、ふだんの個別リスク管理では見落としやすい複合経路が見えやすくなる。  
リバース・ストレステストは、未来を当てるためというより、「自社が何に弱いか」を先に言葉にするための作業だと考えると扱いやすい。

## リスク許容度とリスク戦略はどうつながるのか

リスク管理は、なるべくリスクを小さくする話ではない。  
保険会社はリスクを引き受けて収益を得るので、ゼロにすることは仕事の否定になる。

だから必要なのは、「どのリスクを、どのくらい、どんな条件で引き受けるか」を決めることだ。  
これがリスク許容度やリスク戦略の話である。

たとえば、長期保証商品をどの程度売るのか、外貨建商品をどの水準まで許容するのか、再保険でどこまで移転するのか、ヘッジコストをどこまで払うのかは、全部リスク戦略の問題である。  
数字の計測は、その判断を支える材料であって、ゴールそのものではない。

## 経済価値ベースで何が変わるのか

現行制度では、会計、監督、内部管理で見る尺度が完全にはそろっていない。  
負債はロックイン評価が強く、資産は時価変動の影響を受けやすい。そのため、会計上の純資産の動きと、経済実態としてのサープラスの動きがずれることがある。

経済価値ベースの考え方は、このずれをできるだけ小さくし、資産と負債を同じ物差しで見ようとする。  
そこで重くなるのが、金利感応度、保証とオプション、将来キャッシュフロー、MOCE、ストレス下での耐久力といった論点である。

ただし、2025年度要領では、新規制の具体的計算方法そのものは出題対象外とされていた。  
したがって、第I部でまず押さえるべきなのは、制度の細目暗記よりも、「なぜ経済価値で見ようとしているのか」「従来の何が見えにくかったのか」である。

たとえば現行会計では、超長期の負債が契約時予定利率ベースであまり動かない一方、その他有価証券の債券は時価で揺れる。  
すると、ALM がそれなりに効いていても、会計上は資産だけが先に振れて純資産が大きく上下することがある。経済価値ベースで見たいのは、そうした会計上のずれをならし、金利、株価、スプレッド、保険事故率、解約率、オプション価値の変化まで含めて「本当にどれだけ余力が揺れるのか」を一つの物差しで捉えることである。  
だから経済価値ベースの規制が認識しているリスクは、単に時価評価のぶれではなく、「環境が変わったとき、資産と負債の差額がどのくらい傷みうるか」という会社全体の耐久力の問題だと理解するとよい。

## IAIS の ALM イシューペーパーは、ALM を何のリスク管理と見ているのか

ALM というと金利だけの話に見えやすい。
しかし IAIS の整理では、ALM は `市場リスク` `保険引受リスク` `流動性リスク` をまたいで、資産と負債のズレを管理する話として読まれている。

| リスク類型 | 何がずれると苦しくなるか | 生命保険会社での典型例 |
| --- | --- | --- |
| 市場リスク | 資産価値と負債価値の動きがそろわない | 金利低下で負債価値だけ大きく膨らむ |
| 保険引受リスク | 負債キャッシュフローの形自体が変わる | 解約急増、更新集中、最低保証の重化 |
| 流動性リスク | 今は持っていても、その場で現金を出せない | 大量解約で債券を不利な価格で売る |

この整理が大事なのは、`ALM は運用部門だけの話ではない` ことが見えるからである。
商品設計、契約者行動、流動性準備、再保険、デリバティブまで全部つながってくる。

## IAIS の市場リスク4つは、どう区別するのか

2023-3 の設問で落としたくないのは、`市場リスク = 金利だけではない` という点である。
IAIS に準拠すると、ALM における市場リスクは次の4つで整理できる。

| 市場リスク | 何が変動するか | 会社にとっての不利益 |
| --- | --- | --- |
| 金利リスク | 金利水準や金利感応度 | 資産と負債の現在価値の動きがずれてサープラスが傷む |
| 株式・不動産・その他資産リスク | 市場価格 | 保有資産の下落が負債側の動きと合わず余力が減る |
| 外国為替リスク | 為替レート | 資産と負債の通貨がずれていると評価差が出る |
| 市場リスクに関連する信用リスク | ヘッジ相手先などの信用 | 市場リスク対応の結果、別の信用リスクが立つ |

最後の `市場リスクに関連する信用リスク` は見落としやすい。
たとえば金利リスクをデリバティブで抑えようとしても、相手先が苦しくなれば、そのヘッジ自体が不安定になる。
つまり ALM は、1つのリスクを減らしたら別のリスクが増えることまで含めて見る必要がある。

## 契約者オプションと流動性リスクは、ALM をどう難しくするのか

保険負債の難しさは、金利や株価だけではなく、`契約者が行動を選べること` にもある。
IAIS の ALM イシューペーパーで押さえたい契約者オプションは、少なくとも次の5つである。

- 変額年金等の最低保証オプション
- 契約者貸付オプション
- 保険料超過預入オプション
- 解約オプション
- 契約更新オプション

たとえば、一時払の貯蓄性商品で市場金利が上がると、契約者はより有利な商品へ動きやすくなる。
会社側から見ると、解約が増え、負債キャッシュフローが前倒しされ、換金しにくい資産を売る必要が出るかもしれない。
つまり契約者オプションは、`負債に埋め込まれた選択権` として ALM を難しくする。

流動性リスクの潜在要因も、答案では列挙で切れるようにしておきたい。

| 潜在要因 | 何が危ないのか |
| --- | --- |
| 意図的なミスマッチング戦略 | 平時の利回りを取りにいくほど、必要時の換金性が落ちやすい |
| グループ会社への投資リスク | グループ内資金がすぐ戻らず、自社の流動性を圧迫する |
| 資金調達リスク | 必要時に外部資金を十分確保できない |
| 清算価値リスク | 現金化のタイミングが悪く、含み損が実現損に変わる |

この4つは、結局どれも `必要な時に現金を用意できるか` という一点でつながっている。
生命保険会社は長期負債だから流動性は軽い、と雑に見ると危ない。
平時は持てる資産でも、解約や保険金支払いが前倒しで来れば、時間の余裕が消えた瞬間に苦しくなる。

## 第I部でどう問われるか

リスク管理は、次の5種類で出やすい。

### 1. 基本概念

リスクと不確実性、統合リスク管理、リスク戦略、リスク許容度、3線管理、COSO などである。  
ここは定義だけでなく、何のための枠組みかを押さえておきたい。

### 2. ALM

ALM の目的、デュレーション、コンベクシティ、流動性リスク、契約者オプション、セグメント管理などが中心になる。  
ここは、金利変動時に資産と負債のどちらがどれだけ動くかを、言葉で説明できるようにすると強い。

### 3. ストレステスト

ヒストリカルシナリオ、仮想シナリオ、複数リスク同時変動、リバース・ストレステスト、結果の活用が出やすい。  
単なる用語暗記にせず、「モデルでは拾いにくい部分を補うため」という位置づけを持っておくとよい。

### 4. 監督指針・時事

リスク管理方針、リスクとソルベンシーの自己評価、資産負債の総合的な管理、経済価値ベース規制の三つの柱などが問われる。  
ここは一次資料ベースで現行の言い回しを押さえる必要がある。

### 5. 未出っぽい説明問題

「生命保険会社における ALM の意義を述べよ」のように、過去問の言い回しを少し外した問題でも、問題設定から話せれば崩れにくい。  
この章は、理解がそのまま得点差になりやすい。

## この章の勉強のしかた

順番としては、

1. リスクと不確実性
2. 統合リスク管理の流れ
3. ALM の目的
4. デュレーションとコンベクシティ
5. 流動性リスクと解約オプション
6. ストレステストとリバース・ストレステスト
7. 監督指針と経済価値ベース規制

の流れが入りやすい。

最初の山は、「リスク管理は部門ごとの点検ではなく、会社全体の支払能力と耐久力を守るための経営管理である」というところである。  
ここが見えると、ERM も ALM もストレステストも、それぞれ別の単語ではなく、一つの流れとして見えてくる。
